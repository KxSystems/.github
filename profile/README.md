# KxSystems GitHub Organization

Welcome to the KxSystems GitHub, the home of official open-source repositories from KxSystems.

---

## 🧠 Core Database & Language

Foundational projects for kdb+/q.

* **[kdb](https://github.com/KxSystems/kdb)** - Companion files for kdb+ and q
* **[kdb-taq](https://github.com/KxSystems/kdb-taq)** - Tools and schemas for TAQ-style data in kdb+
* **[kdb-tick](https://github.com/KxSystems/kdb-tick)** - Tickerplant, RDB and pub/sub scripts
---

## 🧩 Language Interfaces

Language bindings for interacting with kdb+.

* **[pykx](https://github.com/KxSystems/pykx)** - Python‑first interface to kdb+/q
* **[embedR](https://github.com/KxSystems/embedr)** - Embed R inside q
* **[javakdb](https://github.com/KxSystems/javakdb)** - kdb+ IPC interface for Java
* **[jdbc](https://github.com/KxSystems/jdbc)** - JDBC client for kdb+
* **[csharpkdb](https://github.com/KxSystems/csharpkdb)** - .NET client library for interfacing with kdb+
* **[ffi](https://github.com/KxSystems/ffi/)** - Foreign Function Interface for kdb+
* **[rkdb](https://github.com/KxSystems/ffi/)** - R client library for kdb+
* **[jupyterq](https://github.com/KxSystems/jupyterq)** - Jupyter kernel for kdb+
* **[kxkdb](https://github.com/KxSystems/kxkdb)** - kdb+ interface for Rust

---

## 🚀 KDB-X Modules

> **KDB‑X** introduced a module framework to `q`. Projects in this section are **incompatible with earlier versions of kdb+**.

* **[logging](https://github.com/KxSystems/logging)** - Logging utilities for KDB-X
* **[printf](https://github.com/KxSystems/printf)** - Formatted printing mirroring `printf` C99 standard
* **[fusionx](https://github.com/KxSystems/fusionx)** - Portable native library wrappers for KDB-X
* **[datagen](https://github.com/KxSystems/datagen)** - Functions for generating synthetic data for various domains
* **[taq](https://github.com/KxSystems/taq)** - Parse NYSE TAQ psv files and persist results in kdb+ format.

---

## 🔗 Integrations

Connectivity with external formats and messaging platforms.

> **KDB-X** has a native parquet integration that is recommended over the parquet read capabilities provided via arrowkdb. [link](https://portal.dl.kx.com/assets/raw/kdb-x/modules/pq/)

* **[arrowkdb](https://github.com/KxSystems/arrowkdb)** - Apache Arrow & Parquet integration
* **[hdf5](https://github.com/KxSystems/hdf5)** - HDF5 file support for kdb+
* **[protobufkdb](https://github.com/KxSystems/protobufkdb)** - kdb+ integration with Google Protocol Buffers
* **[ldap](https://github.com/KxSystems/ldap)** - kdb+ integration with LDAP
* **[mqtt](https://github.com/KxSystems/mqtt)** - MQTT messaging protocol interface
* **[solace](https://github.com/KxSystems/solace)** - Solace middleware client for kdb+
* **[kafka](https://github.com/KxSystems/kafka)** - kdb+ to Apache Kafka adapter

---

## 🧰 ML & NLP

Machine Learning and Natural Language Processing related repositories.

> **KDB-X** has an `ai` module that provides vector database capabilities to kdb+ data stores that is useful for Retrieval Augmented Generation. [link](https://portal.dl.kx.com/assets/raw/kdb-x/modules/ai/)

* **[ml](https://github.com/KxSystems/ml)** - Machine Learning Toolkit
* **[nlp](https://github.com/KxSystems/nlp)** - Natural Language Processing library
* **[automl](https://github.com/KxSystems/automl)** - Automated Machine Learning Framework for kdb+
